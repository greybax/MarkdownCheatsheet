Tasks List renders to

  • If you include a task list in the first comment of an Issue, you will get a handy progress indicator in your issue list. It also works in Pull Requests!

    Example 
    - [x] @mentions, #refs, [links](), **formatting**, and tags supported
- [x] list syntax required (any unordered or ordered list supported)
- [x] this is a complete item
- [ ] this is an incomplete item
Tables renders to

  • Tables are created by adding pipes as dividers between each cell, and by adding a line of dashes (also separated by bars) beneath the header. Note that the pipes do not need to be vertically aligned.

    Example 
    | Option | Description |
| ------ | ----------- |
| data | path to data files. |
| engine | engine to be used for processing templates. |
| ext | extension to be used for dest files. |
Strikethrough renders to

  • Any word wrapped with two tildes (like ~~this~~) will appear crossed out.

    Example 
    ~~this is crossed out text~~
SHA references renders to

  • Any reference to a commit’s SHA-1 hash will be automatically converted into a link to that commit on GitHub.

    Example 
    16c999e8c71134401a78d4d46435517b2271d6ac
mojombo@16c999e8c71134401a78d4d46435517b2271d6ac
mojombo/github-flavored-markdown@16c999e8c71134401a78d4d46435517b2271d6ac
Issue references within a repository renders to

  • Any number that refers to an Issue or Pull Request will be automatically converted into a link.

    Example 
    #1
greybax#1
greybax/MarkdownCheatsheet#1
Username @mentions renders to

  • Typing an @ symbol, followed by a username, will notify that person to come and view the comment. This is called an “@mention”, because you’re mentioning the individual. You can also @mention teams within an organization.

    Example 
    @greybax
Automatic linking for URLs renders to

  • Any URL (like http://www.github.com/) will be automatically converted into a clickable link.

    Example 
    http://www.github.com/
